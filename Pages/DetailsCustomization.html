Details Panel Customization

The Details Panel consists of a set of exposable parameters or functions that you can tweak to edit the properties of your currently selected objects in your Editor Viewport. For example, if you have a static mesh actor selected in your level viewport, the Details panel will allow you to change the materials of the selected actor, edit its transform, or change the mesh itself.

The Details Panel can be completely customized to allow you to override the prebuilt set of properties , add your own properties , or even redesign the panel with your own set of Slate based controls. This can be very useful if you are looking for creating custom editor tools and want to add custom functionalities for specific Actors within your Viewport.

The Details panel Customization is not specific to only actors placed within the level , details panel are also there within your blueprints , static mesh editor and most of the tools that you are familiar with, but that is a discussion for another day. We will get there , no worries.

Default Customization

If you have experience with Unreal C++ you must be aware of UPROPERTY() type variables. When you are creating an Actor class in C++ that needs to be placed in your viewport, you expose several of its variables to be modified via Blueprints or from the Details Panel.

UPROPERTY(EditAnyWhere , BlueprintReadWrite , Category = "Attributes") float MaxStamina;

The EditAnyWhere Specifier allows the property to be edited anywhere, in Blueprints or in level editor Details Panel. The VisibleAnyWhere Specifier shows the property in the Details panel but cannot be directly edited.

To know more about UPROPERTY and various specifiers you can use , you can refer to https://wiki.unrealengine.com/UPROPERTY.

If you want your properties to be editable only if certain conditions are met, you can use EditCondition metadata.

UPROPERTY(EditAnyWhere , BlueprintReadWrite , Category = "Attributes") bool bAllowMagic;
UPROPERTY(EditAnyWhere , BlueprintReadWrite , Category = "Attributes", meta = (EditCondition = "bAllowMagic")) float MaxMana;

To know more about various metadata specifiers you can use , you can refer to https://docs.unrealengine.com/en-us/Programming/UnrealArchitecture/Reference/Metadata.

If you want to add combobox in you Details Panel, you can use Enum Properties.

UENUM(BlueprintType) enum class ECharacterClass : uint8 { Mage UMETA(DisplayName = "Mage"), Warrior UMETA(DisplayName = "Warrior"), Rogue UMETA(DisplayName = "Rogue") };

When you create a Blueprint Actor and implement your logic within your construction script, you use Editable variables to be able to tweak from your level editor Details panel. However, any tweaking would require the entire construction script to be executed, thus increasing your compile time. However, creating the base Actor class in C++ has an additional advantage of executing only the part of the code which is directly affected by changing the values of a particular editable variable. To implement this, we override the PostEditChangeProperty function in our Actor class.

#if WITH_EDITOR virtual void PostEditChangeProperty(FPropertyChangedEvent& PropertyChangedEvent)override; #endif

Default Customizations via UPROPERTY() should be enough most of the time and you may never need further customizations unless you are creating some specific tools where additional controls like a button or dynamic combobox can greatly surve your specific requirements. My main purpose of this article is to deep dive the details of how exactly you can go along with customizing your Details panel to suit your specific requirements, with step by step examples to help you catch up with the process.

Getting Started

We need two classes to get started with Details Panel Customization, the UObject class containing the properties that are to be exposed for customizations, and the DetailCustomization class, where we are implementing our customization logic.

Let us create an Actor class that we can spawn in our level, with few UProperties. Go to File -> New C++ Class , and choose Actor as the parent class.

UPROPERTY(EditAnyWhere , BlueprintReadWrite , Category = "Attributes") float MaxStamina;
